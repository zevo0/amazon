# هيكل مشروع أمازون شاليه — التوثيق الكامل

هذا الملف يشرح شجرة المشروع، وظيفة كل ملف، والمنطق اللي يشتغل عليه الموقع، عشان يكون مرجع لك ولأي حد يشتغل على المشروع بعدك.

---

## 2. الفكرة الأساسية (المنطق العام)

الموقع مبني بـ **HTML/CSS/JS خام** بدون أي إطار عمل (لا React ولا Vue) — يعني أي متصفح يفتحه مباشرة بدون خطوة "بناء" (build). هذا هو سبب توافقه مع أي استضافة ثابتة (Vercel، Cloudflare Pages، أو حتى استضافة عادية).

**الفصل بين الشكل والبيانات:**
- كل التصميم (الألوان، الخطوط، الحركات، شكل الأزرار) موجود في `css/style.css` و `admin/css/admin.css` فقط.
- كل الاتصال بقاعدة البيانات يمر من نقطة وحدة: `js/supabase-client.js`. هذا يعني أي تغيير مستقبلي في طريقة تخزين البيانات (Supabase، أو أي شي ثاني) ما يحتاج يلمس التصميم إطلاقًا.

**وضعان يشتغل عليهم الموقع:**
1. **الوضع التجريبي (Demo Mode):** إذا `SUPABASE_URL` في `supabase-client.js` لسه فيها القيمة الافتراضية، الموقع يشتغل ببيانات محلية جاهزة (مكتوبة داخل `main.js` و `admin.js`) — يفتح ويشتغل بس ما يحفظ أي تعديل بشكل دائم.
2. **الوضع الحي (Live Mode):** بعد ما تحط رابط ومفتاح Supabase الحقيقيين، كل قراءة وكتابة تصير فعلية من/إلى قاعدة البيانات، ولوحة التحكم تصير تحفظ كل شي بشكل دائم.

الكود يكتشف الوضع تلقائيًا (دالة `isSupabaseReady()`) — ما تحتاج تبدّل أي شي ثاني غير السطرين في `supabase-client.js`.

## 3. منطق الصفحة الرئيسية (`index.html` + `js/main.js`)

### أ. المميزات + المعرض (قسم واحد مدمج)
كل ميزة (مسبح، جلسات خارجية، غرف نوم...) لها معرض صور خاص فيها. البيانات تجي من دالة `fetchFeatures()` اللي تجيب كل ميزة مع صورها المرتبطة فيها (جدول `feature_images` مربوط بـ `features` عبر `feature_id`).

### ب. التقويم والحجز
- كل يوم مستقبلي **متاح افتراضيًا** إلا لو الأدمن غيّره يدويًا من اللوحة — ما فيه توليد عشوائي للحالات.
- عند اختيار يوم، تفتح نافذة فيها **بطاقة اليوم** (صورة + التاريخ + الحالة)، وقائمة الباقات.
- **سعر عطلة نهاية الأسبوع:** لو اليوم جمعة أو سبت، يضاف ١٥ ر.ع تلقائيًا على كل باقة (`WEEKEND_SURCHARGE` في `main.js`).
- **العربون:** يُحسب تلقائيًا — ٤٠٪ لو الحجز بعد أكثر من ٣ أيام، أو المبلغ كامل لو أقل من ٣ أيام.
- عند الضغط "تأكيد الحجز"، يفتح واتساب برسالة جاهزة فيها كل التفاصيل (التاريخ، الباقة، السعر، المبلغ المطلوب).

### ج. الأوقات (بدون قيم ثابتة بالكود)
كل باقة لها `start_time` و `end_time` (مثلاً `08:00` و `13:00`)، ودالة `formatTimeRange()` في `time-utils.js` تحوّلهم تلقائيًا لصيغة عربية (`٨ص - ١ظ`). لما تعدّل الوقت من لوحة التحكم، يظهر التحويل الجديد مباشرة بدون ما تلمس الكود.

### د. الفوتر
رقم الهاتف يُقرأ من نفس المتغير `WHATSAPP_NUMBER` المستخدم بزر واتساب، ويُعرض بدون رمز الدولة (`968`) تلقائيًا.

## 4. منطق لوحة التحكم (`admin/index.html` + `admin/js/admin.js`)

### أ. صفحة واحدة، لا تبديل
كل الأقسام (نظرة عامة، الصور، المميزات، الباقات، التقويم، الموقع، الإعدادات) معروضة في نفس الصفحة فوق بعض. الشريط العلوي فيه روابط "قفز" (`<a href="#sec-...">`) تنقلك لمكان القسم بحركة ناعمة، بدون إخفاء أي قسم ثاني.

### ب. تسجيل الدخول
- **وضع تجريبي:** أي إيميل وكلمة مرور تدخلك (عشان تجرب الواجهة).
- **وضع حي:** يستخدم Supabase Auth الحقيقي — لازم تنشئ المستخدم من لوحة Supabase (Authentication → Users).

### ج. كل قسم وربطه بقاعدة البيانات

| القسم | يعدّل جدول | ملاحظة |
|---|---|---|
| الصور | `gallery` | يرفع للـ Storage bucket اسمه `gallery` |
| المميزات | `features` + `feature_images` | كل ميزة معرض صور خاص بها |
| الباقات | `packages` | السعر + وقت البداية/النهاية، يُحفظ فور التعديل |
| التقويم | `availability` | تبديل يدوي لكل فترة (صباحي/مسائي/يوم كامل/مبيت) لكل يوم |
| الموقع | `settings` | اسم الموقع، الوصف، رابط الخريطة |
| الإعدادات | `settings` | رقم واتساب، رابط انستقرام، صورة الغلاف |

كل تعديل بلوحة التحكم يستدعي دالة مطابقة في `supabase-client.js` (مثل `supabaseClient.from('packages').update(...)`) — ونفس التعديل ينعكس فورًا بالواجهة المحلية، وبشكل دائم في قاعدة البيانات لو الوضع حي.

## 5. أشياء لازم تعرفها وتتابعها

- **الخط:** الموقع جاهز لاستخدام خط "Tinta Arabic" لو اشتريت رخصته التجارية (الملف يحتاج `.woff2` توضعه في `assets/fonts/`) — حاليًا يستخدم خط بديل أنيق تلقائيًا لأن الخط الأصلي مو مرخّص للاستخدام التجاري.
- **رابط الموقع المؤقت:** فيه رابط placeholder (`amazon-chalet.vercel.app`) بثلاث أماكن — `index.html` (سطرين: canonical و og:url) و `sitemap.xml`. لازم تبدله برابط موقعك الفعلي بعد النشر.
- **رقم الواتساب:** لسه فيه رقم تجريبي `96890000000` في `js/main.js` — لازم تبدله برقمك الحقيقي.
- **رابط الخريطة:** موقع تقريبي (جبل الأخضر) داخل `index.html` قسم `#location` — بدّله بموقع الشاليه الفعلي من Google Maps.
- **الاستضافة:** الموقع ثابت بالكامل (Static) — يشتغل على Vercel أو Cloudflare Pages بدون أي خطوة بناء، طالما البيانات جايه من Supabase (وليس PHP/MySQL).
